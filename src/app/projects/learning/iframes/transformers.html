Self-Attention Explained

A developer's guide to the core mechanism of Transformers.

Step 1: Input & Embeddings

The input sentence is first tokenized and each token is converted into a numerical vector (an embedding).

The
[0.1, 0.5, -0.2]
developer
[-0.4, 0.8, 0.1]

the

code
[0.2, 0.1, -0.5]

Step 2: Creating Q, K, V Vectors

Each embedding is linearly transformed into three new vectors: a Query (Q), a Key (K), and a Value (V) vector. These are like a search query, a search tag, and the content itself.

developer
Query (Q)
[0.5, 0.1, -0.3]
code
Key (K)
[0.2, -0.1, 0.4]
Value (V)
[0.8, 0.6, 0.9]
↓
This is done for ALL tokens in the sentence.

Step 3: Scoring & Attention Weights

The `Query` vector of each token is multiplied by the `Key` vector of every other token to get a similarity score. This score is then normalized using a Softmax function to get a final weight (a number between 0 and 1).

The values above are simplified raw scores. These are then normalized with Softmax to get final attention weights that sum to 1 for each row.

Step 4: The Final Output Vector

Each token's new, context-aware vector is a weighted sum of **all** the `Value` vectors, where the weights are the scores from the previous step. This is how the model "attends" to different parts of the sentence.

Value Vectors
[0.8, 0.6, 0.9] (from "code")
[...] (from "wrote")

Weights
0.8 (from "developer" → "code" score)
0.1 (from "developer" → "wrote" score)
↓
New Vector for "developer"
[0.75, 0.5, 0.8] (example)

This new vector for "developer" now contains information from all other words, making it context-aware.
